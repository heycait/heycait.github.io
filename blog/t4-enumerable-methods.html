    Enumerable

    Enumerable is a type of module in Ruby and is similar to classes. It's use is to collect related methods and constants, such as the Math module which includes math constants and operations like Pi (Math::Pi) and square root (Math.sqrt(#)). However, modules can also be used as mixins which allows for the use of properties outside a class. Since Ruby does not allow for multiple inheritance like other object-orienting languages, using a module as a mixin is an alternative to incorporating other methods and definitions outside the class you're working with.

    In the case of the Enumerable module, that gives us the use of methods that search and sort. As such, it appears to be applicable only for ranges, arrays, and hashes.

    Reference

    Enumerable method: group_by

    group_by {|object| code block} --> result_hash
    group_by --> an_enumerator

    The method group_by sorts a collection according to the results of a code block (if there is no code block, an enumerator is return). The collection is returned as a hash where the keys are the results from running the code block and the values are an array of the elements in the collection that result in that key. For example:

    (1..6).group_by {|number| number % 3}
    #=> {0 => [3,6], 1 => [1,4], 2 => [2,5]}

    The code block is asking for the results of taking an number from 1 to 6, dividing it by 3, and finding the remainder. This returns a hash that organizes the numbers 1 to 6 by the remainder it returns when evaluating the expression (number % 3). Reading the hash, we know that 3 and 6 returns no remainder (a value of 0), 1 and 4 returns a remainder of 1, and 2 and 5 return a remainder of 2.

    NOTE: While this method can be used on ranges, arrays, and hashes, it seems most suitable to ranges and arrays with relatively simple code blocks.

    Uses

    The Enumerable group_by method is very useful if you want to organize data by the results of a code block. It can be used for grouping numerical answers to a mathematical expression such as the example shown above or it could be used to group other data such as strings by some property. Although it's possible to use this method to sort strings, it appears the best and ideal usage is for numbers and mathematical expressions with simple code blocks (where you want the results of different answers, if you're only looking for one answer, the select method would be better).

    There are many ways of using this method but other methods exist that would be better still. Unfortunately, I can't think of any example besides the one shown above that would utilize this method the best and be the simplest solution.
