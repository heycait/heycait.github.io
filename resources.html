    • Teach Yourself to Program

    Have you wanted to learn programming but never knew where to start? What does cd mean? What is git? What is rspec? Node? Why don't I know what's going on?! If you've ever had any of these questions or thoughts, it can seem overwhelming and frustrating because you don't know what you don't know so you have no clue what you need to know. Well that's where these resources can come in handy. As someone who had virtually no programming experience prior to applying for Dev Bootcamp and App Academy, there were many times when I wanted to quit because I didn't know anything and couldn't figure it out. Lucky you, you've got me to guide you.

    I'll add more resources as I come across them but in the mean time, if you're a complete newbie, here's what you should learn first to avoid confusion and misunderstandings and create and build up your online presence (if you choose to pursue this professionally).

    1. The Terminal/Command Line
    2. Git & GitHub (optionally but recommended)

    Besides learning the command line and git/GitHub, I would suggest going through Codecademy's courses and other online tutorials like General Assembly's Dash or Ruby Monk (if you want to learn Ruby). If you choose to do Ruby Monk, do it after going through Codecademy's Ruby track, otherwise it WILL confuse the hell out of you.

    I personally suggest learning HTML/CSS first and then move into other languages. Items 4/5 can be learned in either order. For number 5, any other language would do but I think Ruby is really beginner friendly.

    1. HTML/CSS
    2. JavaScript
    3. jQuery
    4. SQL
    5. Ruby/Python [avoid PHP]

    For those choosing to learn Ruby, I would also read these books (some are free) in this exact order. Well POODR could be read after Beginning Ruby if you prefer.

    1. Learn to Program by Chris Pine
    2. Beginning Ruby From Novice to Professional by Peter Cooper
    3. The Well-Grounded Rubyist (WGR) by David A. Black
    4. Practical Object-Oriented Design in Ruby (POODR) by Sandi Metz

    If you want to do Test First Ruby's problem set, I highly recommended getting through WGR first before attempting. At least get through the first 10 chapters of both Beginning Ruby and WGR.
